# 2026-05-15 视频分析 Prompt 优化

## 问题描述

视频分析结果全是"免责声明"式废话，例如：
- "视频画质一般，侧面视角导致关键跳跃阶段与刃型难以辨识。低置信度帧较多，结果仅供参考。"
- "视频画质模糊且拍摄角度受限，无法清晰辨识旋转动作细节。低置信度帧较多，结果仅供参考。"

分析了半天，**看不到任何实质性技术结论**。

## 根因分析

三层"防御性"逻辑叠加，导致 AI 不敢输出任何实质性分析：

1. **System Prompt 预设悲观基调**：`vision_prompt_templates.py` 的 system prompt 告诉 AI "视频可能是侧面、斜角、远距离或低清晰度"，AI 倾向于直接放弃而不是尽力分析。
2. **Report Prompt 主动要求输出免责声明**：`report.py` 中明确指令"如果 low_confidence_frame_count 大于 0，请在 summary 中明确提醒'低置信度帧较多，结果仅供参考'"。
3. **HIGH_CONF_THRESHOLD = 0.5 过高**：训练视频的帧 confidence 普遍在 0.3~0.5 之间，0.5 的阈值导致大部分帧被归为"低置信度"，触发免责声明逻辑。

恶性循环：悲观 system prompt → AI 给低 confidence → 高置信帧不足 → report prompt 要求输出"仅供参考" → 用户看不到分析结果。

## 修改内容

### 1. 降低高置信度阈值（建议 1）

**文件**：`backend/app/services/report.py`

```diff
-HIGH_CONF_THRESHOLD = 0.5
+HIGH_CONF_THRESHOLD = 0.3
```

**效果**：训练视频的帧不再被大面积判定为"低置信度"，`summarize_vision_for_report` 中 `reliable_frames` 包含更多帧，`low_confidence_frame_count` 减少，免责声明触发频率大幅降低。

### 2. 弱化 System Prompt 的悲观引导（建议 4）

**文件**：`backend/app/services/vision_prompt_templates.py`

**修改前**：
```
- 视频可能是侧面、斜角、远距离或低清晰度。
- 如果脚踝、冰刀或入跳弧线不可见，不要强行判断刃型。
- 如果证据不足，请输出"不可判断"并降低 confidence。
```

**修改后**：
```
- 请尽力从可见信息中提取分析结论，即使视角受限也尝试给出观察。
- 刃型判断：如果脚踝、冰刀或入跳弧线可见，给出具体刃型；如果确实不可见，再标记"不可判断"并简要说明原因。
- 不确定时可以降低 confidence，但不要因为不确定就直接放弃分析。
```

**效果**：AI 从"预设放弃"变为"尽力分析"，即使视角受限也会尝试给出观察。

### 3. 移除 Report Prompt 的免责声明指令（建议 3）

**文件**：`backend/app/services/report.py`

**修改前**：
```
如果 low_confidence_frame_count 大于 0，请在 summary 中明确提醒"低置信度帧较多，结果仅供参考"，
并避免过度肯定的结论。如果 fallback_to_all_frames 为 true，请指出高置信帧不足。
```

**修改后**：
```
基于 available frames 生成实质性分析，给出具体的技术问题和改进建议。
如果数据确实不足，在 data_quality 字段中标记即可，不必在 summary 中反复强调数据质量问题。
```

**效果**：Report 模型不再被指令输出"仅供参考"，而是被要求输出实质性分析。

## 涉及文件

| 文件 | 修改类型 | 说明 |
|------|----------|------|
| `backend/app/services/report.py` | 阈值 + Prompt | 降低 HIGH_CONF_THRESHOLD，移除免责声明指令 |
| `backend/app/services/vision_prompt_templates.py` | System Prompt | 从"预设悲观"改为"鼓励尽力分析" |

## 预期效果

- 分析结果中出现**具体的技术问题和改进建议**，而非空洞的免责声明
- `data_quality` 字段正确反映数据质量，但不在 summary 中反复强调
- AI 在视角受限时仍会尝试给出观察，而不是直接输出"不可判断"

## 回滚方式

如需回滚，将 `HIGH_CONF_THRESHOLD` 改回 `0.5`，并恢复原 system prompt 和 report prompt 文本。
