## 二、当前性能指标推测

以下指标基于代码实现逻辑和典型花滑训练视频特征推测，非真实测试数据。

### 2.1 Profile 推断准确率

| Profile | 推断依据 | 推测准确率 | 主要误差来源 |
|---------|---------|-----------|-------------|
| jump | CoM 垂直范围 + 运动密度 + 腾空帧 | ~85-90% | 近距离拍摄时 CoM 范围小，可能误判 |
| spin | 髋部旋转信号 | ~75-80% | 侧面拍摄时旋转信号弱 |
| step | 用户输入 + 几何验证 | ~90-95% | 几乎纯依赖用户输入 |
| spiral | 螺旋线门控（低运动 + 低波动） | ~70-75% | 与低速步法混淆 |

### 2.2 跳跃种类识别准确率

当前系统**不做跳跃种类自动识别**（Axel/Lutz/Flip/Loop/Salchow/Toe Loop），仅通过 LLM 视觉分析给出初步判断。v1.1.7 新增了弱几何证据辅助。

| 跳跃种类 | LLM 识别难点 | v1.1.7 几何证据辅助 | 推测准确率 |
|---------|-------------|-------------------|-----------|
| Axel | 前向起跳特征明显 | approach_direction=forward 强信号 | ~60-70% |
| Lutz vs Flip | 需区分外刃/内刃，480p 下困难 | pre_takeoff_edge_score 辅助，但置信度低 | ~35-45% |
| Loop | 双腿并拢特征明显 | feet_together_at_takeoff 强信号 | ~55-65% |
| Salchow | 自由腿摆动特征 | free_leg_swing_amplitude 辅助 | ~55-65% |
| Toe Loop | 点冰特征 | toe_pick_pulse 辅助 | ~55-65% |

**混淆矩阵推测**（主要混淆对）：
- Lutz ↔ Flip：最高混淆率（~40-50% 互相混淆），因仅靠刃型区分
- Loop ↔ Toe Loop：中等混淆率（~20-30%），因都从后外刃起跳
- Axel 较少与其他混淆（前向起跳特征独特）

### 2.3 跳跃周数识别准确率

当前实现基于 `转速 × 滞空时间` 估算周数，精度受以下因素影响：

| 周数 | 推测准确率 | 说明 |
|------|-----------|------|
| 单跳（1T/1S 等） | ~80-85% | 滞空时间短，转速低，容易区分 |
| 双跳（2A/2T 等） | ~65-70% | 与单跳/三跳边界模糊 |
| 三跳（3Lz/3T 等） | ~45-55% | 误差累积导致周数判断不稳定 |
| 四跳（4T 等） | ~30-40% | 极高速旋转，抽帧率不足 |

**不足周/足周/过周判断**：
- 当前系统不做此判断（仅估算总周数）
- 估算精度约 ±0.5 圈
- 对于 2A（2.5 圈）vs 3T（3 圈）边界情况区分困难

### 2.4 旋转定级一致率

当前系统**不做旋转定级**（如 USFS 的 I/II/III/IV 级别），仅评估：
- 旋转轴稳定性（average_tilt_degrees → stability_score）
- 转速（rotation_rps）

与 ISU/USFS 裁判定级的对应关系未建立。推测若强行映射，一致率较低（~30-40%）。

### 2.5 动作时间边界误差

| 边界 | 抽帧率 | 每帧间隔 | 推测误差 |
|------|--------|---------|---------|
| 起跳帧（T） | 10fps (jump) | 100ms | ±1 帧（±100ms） |
| 顶点帧（A） | 10fps | 100ms | ±1 帧（±100ms） |
| 落冰帧（L） | 10fps | 100ms | ±1-2 帧（±100-200ms） |
| 旋转入/出 | 8fps (spin) | 125ms | ±2-3 帧（±250-375ms） |

注：v1.1.9 起 jump 使用 10fps 采样（原 5fps），时间精度提升一倍。

### 2.6 整体动作序列准确率

| 维度 | 推测准确率 | 说明 |
|------|-----------|------|
| 动作分割（phase detection） | ~70-80% | LLM 视觉分析 + phase_smoother 约束 |
| 动作分类（profile inference） | ~85-90% | 几何门控（jump/spin/step/spiral 四分类） |
| 联合准确率（分割+分类） | ~60-70% | 两阶段误差累积 |
| 问题检出率 | ~75-85% | LLM 对明显问题检出率高，细微问题易遗漏 |
| 误报率 | ~10-20% | LLM 偶尔"幻觉"出不存在的问题 |

## 三、关键瓶颈与改进方向

### 3.1 最大瓶颈：LLM 视觉分析的不确定性

当前系统的视觉分析完全依赖多模态 LLM（v1.1.10 起增加了自一致投票，v1.1.11 起增加了多 provider 投票），但仍存在：
- **不确定性**：同一视频多次分析可能得到不同结果
- **幻觉**：LLM 可能"看到"不存在的问题
- **精度限制**：480p + 10fps 的输入质量限制了分析精度
- **成本**：每次分析约 32 帧 × ~400 tokens × 2 路 = ~25000 tokens

**改进方向**：引入专用动作识别模型（SlowFast/VideoMAE）替代 LLM 做 phase detection，LLM 仅用于生成文字报告。

### 3.2 第二大瓶颈：慢动作视频处理

v1.1.2 已修复 effective_fps 问题，但仍存在：
- 240fps 源视频经 10fps 采样后，每帧间隔实际是源视频的 24 帧
- 关键姿态可能落在采样间隙中

**改进方向**：自适应抽帧率（根据运动密度动态调整 fps）。

### 3.3 第三大瓶颈：关键点抖动

v1.1.6 已加入 One-Euro Filter 平滑，但高速旋转场景下仍有抖动。

**改进方向**：增加 Kalman Filter 或 Transformer 时序模型。

### 3.4 缺失能力

| 能力 | 当前状态 | 难度 |
|------|---------|------|
| 跳跃种类自动识别 | 未实现（依赖 LLM 猜测 + 弱几何证据） | 中 |
| 跳跃周数精确识别 | 粗糙估算（±0.5 圈） | 中 |
| 旋转定级 | 未实现 | 高 |
| 刃型检测（内刃/外刃） | 未实现（弱几何证据置信度低） | 高 |
| 连跳分段 | 未实现 | 中 |
| 自由滑节目动作序列标注 | 未实现 | 高 |
| 不足周/足周/过周判断 | 未实现 | 中 |
