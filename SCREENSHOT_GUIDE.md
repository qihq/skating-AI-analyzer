# Screenshot Guide

## Recommended GitHub Screenshots

Use 3 to 5 screenshots so visitors can understand the product quickly.

## Shot 1: Skill Tree

- Page: `/path`
- Goal: show the child-friendly skill tree and the highlighted "Now Practicing" section
- Include:
  - skill cards
  - stage progress
  - focus area at the top

Suggested caption:
`Skill tree with active practice focus and stage-based progression.`

## Shot 2: Review Upload Flow

- Page: `/review`
- Goal: show upload, progress steps, and structured review form
- Include:
  - selected video area
  - progress bar or status steps
  - skater / skill selection

Suggested caption:
`Training video upload and analysis workflow with progress tracking.`

## Shot 3: Report Page

- Page: `/report/:id`
- Goal: show the generated analysis report and scoring presentation
- Include:
  - overall score
  - structured report sections
  - biomechanics or issue summaries

Suggested caption:
`AI-generated report with score, movement feedback, and training suggestions.`

## Shot 4: Archive / Progress

- Page: `/archive`
- Goal: show timeline, historical sessions, and progress visibility
- Include:
  - archive list or grouped timeline
  - score badges
  - child / parent mode context

Suggested caption:
`Archive timeline for tracking progress across training sessions.`

## Shot 5: Settings / Provider Management

- Page: `/settings`
- Goal: show the operational side of the product
- Include:
  - provider configuration
  - backup or PIN management
  - system settings

Suggested caption:
`Operational settings for AI providers, backups, and parent controls.`

## Capture Tips

- Prefer desktop width around 1440px
- Keep sensitive data hidden
- Use realistic but non-private sample content
- Avoid exposing API keys, PIN values, or local file paths

## Recommended Order for README

1. Skill tree
2. Upload / review flow
3. Report page
4. Archive / progress
